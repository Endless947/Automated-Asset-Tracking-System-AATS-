PROJECT OVERVIEW
=================
This project is called **AATS – Automated Asset Tracking System**.
Think of it like a smart security guard for computer labs:
- It **watches important devices** (USB mouse, keyboard, Bluetooth headset, etc.).
- It **notices when something is unplugged or walks away**.
- It **sends messages** to a central server.
- The server **thinks for a bit** (debounce time).
- If the problem is real (not just a quick bump), it **raises a loud red alert** on the admin dashboard and saves everything to a database.

The project has three main parts:
- `student_agent/` – a small program that runs on each lab PC and watches devices.
- `server/` – a FastAPI backend that receives MQTT messages, stores data in SQLite, and applies alert logic.
- `admin_dashboard/` – a web page that shows live status and history.

Below we explain **every file** and, inside each file, **every line or group of lines**, in simple (kid-friendly) language but with enough detail for experienced programmers.

Severity helper:

L59–L64: `def severity_for(status: str) -> str:`
- Maps raw device `status` to a normalized severity:
  - `CONNECTED` → `OK`.
  - `WEAK_SIGNAL` → `WARNING`.
  - Everything else (e.g. `MISSING`) currently maps to `WARNING`.

Timeout helper:

L67–L72: `def timeout_for(payload: Dict) -> int:`
- Chooses debounce timeout seconds based on:
  - If `device_type` is `"usb"` and `status` `"MISSING"` → use USB timeout.
  - If `device_type` is `"bluetooth"` and status is `"MISSING"` or `"WEAK_SIGNAL"` → use Bluetooth timeout.
  - Otherwise return 0 (no debounce, treat directly).

Status handler:

L75–L77: `def handle_status(payload: Dict) -> None:`
- Writes PC heartbeat info into SQLite by calling `db.upsert_heartbeat`.

Event handler (core logic):

L79–L97: `def handle_event(payload: Dict) -> None:`
- L80: Build key `(lab_id, pc_id, device_id)`.
- L81: Read raw `status`.
- L83–L97: If status is `"CONNECTED"`:
  - L84–L86: Remove any pending entry and store it in `pending_event`.
  - L87–L88: Upsert device state as `OK` and `CLOSED`, insert an `OK` event.
  - L90–L96: If there was a pending event, insert a synthetic **glitch resolved** warning event, including details about when the pending window started.
  - L97: Return; nothing else to do.

L99–L103: Non-CONNECTED events.
- L99: Compute `timeout_sec = timeout_for(payload)`.
- L100–L103: If `timeout_sec <= 0` (no debounce needed):
  - Upsert device state with severity based directly on status and `alert_status="OPEN"`.
  - Insert an event with same severity and `OPEN`.
  - Then return.

L105–L117: Debounced path (timeout > 0).
- L105–L113: Under `_pending_lock`:
  - If the key is not in `pending`, add it with:
    - `started_at` (current `time.time()`),
    - `timeout_sec`,
    - the original `payload`,
    - `confirmed=False`.
  - Whether new or existing, get `pending_state`.
  - Compute `pending_since` as human‑readable ISO string from `started_at`.
- L116: Upsert device state as:
  - `severity="WARNING"`,
  - `alert_status="PENDING"`,
  - with `pending_since` timestamp.
- No DB event is inserted yet; that only happens when timeout is confirmed.

Pending watcher thread:

L119–L137: `def pending_watcher() -> None:`
- Infinite loop that:
  - L121: Gets current `time.time()`.
  - L122: Local list `to_confirm`.
  - L124–L131: Under lock, iterate through pending items:
    - Skip already confirmed items.
    - If `now - started_at >= timeout_sec`, mark `confirmed=True` and add to `to_confirm`.
  - L132–L135: For each item in `to_confirm` (outside lock):
    - Extract `payload`.
    - Upsert state as `severity="CRITICAL"`, `alert_status="OPEN"`, `pending_since=None`.
    - Insert a `CRITICAL` event with `details={"debounce_seconds": timeout_sec}` so reports know how long it waited.
  - L137: Sleep 1 second and repeat.

L140–L150: `@app.on_event("startup") def on_startup() -> None:`

L153–L157: `@app.on_event("shutdown") def on_shutdown() -> None:`

- Takes `lab_id`, `pc_id`, `pc_status`, `last_seen`, `agent_version` from payload.

  - `updated_at` with `now_iso()`.

L140–L167: `insert_event(self, payload, severity, alert_status, details=None)`
- Inserts a row into `device_events`.
- Uses `INSERT OR IGNORE` so repeated events with same unique `event_id` are not duplicated.

- Returns current UTC ISO string.

- Async function to scan Bluetooth devices using `BleakScanner.discover`.

- If callback provided, calls it with payload.

  - RSI less than weak threshold → `"WEAK_SIGNAL"`.

L65–L69: Early return if `BleakScanner` is None.
- Prints message that Bluetooth monitor is disabled.

- While running:
  - Tries to run async `_scan_devices()` using `asyncio.run`.
  - If an error occurs, prints error and uses empty result.
  - For each configured device:
    - Read `device_id`, MAC (uppercased), alias, weak_threshold (default −75).
    - Lookup RSSI from `seen` dict.
    - Determine new status via `_resolve_status`.
    - Compare to `_state`; if changed:
      - Update `_state`.
      - Call `_emit` with payload including device details, RSSI, observed_at, and `source="bluetooth_monitor"`.
  - Sleep `poll_interval_sec`.
